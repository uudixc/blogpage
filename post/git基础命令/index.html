Git基础命令

Git是一种版本控制系统，用于跟踪代码的更改历史记录，协作开发，以及管理项目。

以下是基本的Git使用方法：

  • 安装Git：下载并安装Git客户端，可以从Git官方网站 https://git-scm.com/downloads 或者其他渠道获得。
  • 初始化Git仓库：在本地项目目录中打开命令行或终端，输入“git init”命令，将创建一个Git仓库。
  • 添加文件到Git仓库：使用“git add”命令将文件添加到Git仓库。例如，要添加一个名为“myfile.txt”的文件，可以运行“git add myfile.txt”。
  • 提交更改：使用“git commit”命令提交更改到Git仓库。例如，运行“git commit -m ‘added myfile.txt’”将添加一个提交消息并提交更改。
  • 查看更改历史记录：使用“git log”命令查看Git仓库的更改历史记录。例如，运行“git log”将显示每个提交及其提交消息。
  • 创建分支：使用“git branch”命令创建分支。例如，要创建名为“mybranch”的分支，可以运行“git branch mybranch”。
  • 切换分支：使用“git checkout”命令切换到另一个分支。例如，要切换到名为“mybranch”的分支，可以运行“git checkout mybranch”。
  • 合并分支：使用“git merge”命令将一个分支合并到当前分支。例如，要将名为“mybranch”的分支合并到当前分支，可以运行“git merge mybranch”。
  • 推送到远程仓库：使用“git push”命令将更改推送到远程Git仓库。例如，要将更改推送到名为“origin”的远程仓库，可以运行“git push origin”。
  • 克隆远程仓库：使用“git clone”命令克隆远程Git仓库到本地。例如，要克隆名为“myrepo”的远程仓库，可以运行“git clone git://github.com/user/myrepo.git”。
  • 拉取最新更改：使用“git pull”命令从远程Git仓库拉取最新更改。例如，要从名为“origin”的远程仓库拉取最新更改，可以运行“git pull origin”。
  • 忽略文件：可以创建名为“.gitignore”的文件，列出不需要被Git跟踪的文件或文件夹。例如，要忽略所有“.log”文件，可以在“.gitignore”文件中添加“*.log”。
  • 撤销更改：使用“git reset”命令可以撤销已经提交到Git仓库中的更改。例如，要撤销最新的提交并保留更改内容，可以运行“git reset HEAD~1”。
  • 暂存更改：使用“git stash”命令可以暂存未提交的更改，以便稍后再使用。例如，要暂存所有未提交的更改，可以运行“git stash save”。
  • 恢复暂存的更改：使用“git stash apply”命令可以恢复先前暂存的更改。例如，要恢复最新的暂存更改，可以运行“git stash apply”。
  • 查看更改差异：使用“git diff”命令可以查看更改的差异。例如，要查看当前工作区和最新提交之间的差异，可以运行“git diff HEAD”。
  • 查看文件历史记录：使用“git blame”命令可以查看每个行的修改历史记录。例如，要查看名为“myfile.txt”的文件的历史记录，可以运行“git blame myfile.txt”。
  • 操作标签：使用“git tag”命令可以创建、查看和删除标签。例如，要创建名为“v1.0”的标签并附上提交消息，“git tag -a v1.0 -m ‘release v1.0’”。
